## Featured Projects

### [italy-job-hunter](https://github.com/simonecamerano/italy-job-hunter)
AI-powered job hunting automation for the Italian tech market.

A multi-model pipeline that searches the web, runs boolean AI triage, matches against a CV, and delivers formatted reports to Telegram — fully automated, zero manual intervention.

```
Tavily (search) → Groq/llama-3.1 (triage) → DeepSeek-V3 (CV analysis) → Telegram
```

`Node.js` `Tavily` `Groq` `DeepSeek` `Docker` `Vitest`

---

### [contextforge](https://github.com/simonecamerano/contextforge)
Local-first project memory engine for developers and AI agents.

Scans a codebase and builds a structured, version-controlled memory layer in `.contextforge/` — architecture, active context, technical decisions, open questions. Queryable from the terminal or injectable as a precise context window for any LLM.

`TypeScript` `CLI` `LLM-agnostic` `Ollama` `DeepSeek` `Vitest`

## How I Think About Problems

Most developers ask *"how do I build this?"*  
I start with *"what process breaks here, and what's the minimal system that fixes it reliably?"*

That's the difference 26 years of operational experience makes.

AI is infrastructure — not a feature. I use it to build systems that would be impossible or impractical to build manually, not to add a chatbot to something that doesn't need one.
